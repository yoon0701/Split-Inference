# Split Inference with PyTorch

이 프로젝트는 PyTorch를 사용하여 딥러닝 모델(MobileNetV2)을 두 부분으로 나누어 클라이언트와 서버에서 각각 실행하는 분산 추론(Split Inference)을 구현합니다. 클라이언트(젯슨 보드 등 엣지 디바이스)는 모델의 앞부분을, 서버는 모델의 뒷부분을 담당하여 엣지 디바이스의 연산 부하를 줄이는 것을 목표로 합니다.

---

## 📁 파일 구성 및 설명

### 1. `server.py`
* **역할**: 🖥️ **서버 프로그램**
* **주요 기능**:
    * 분할된 딥러닝 모델의 뒷부분(`part2`)을 로드합니다.
    * 클라이언트(`jetson.py`)로부터 중간 결과물(텐서)을 TCP/IP 소켓 통신으로 수신합니다.
    * 수신한 텐서로 나머지 추론을 수행하여 최종 결과를 계산합니다.
    * 최종 예측 결과를 다시 클라이언트로 전송합니다.
* **실행 방법**:
    ```bash
    python server.py
    ```

### 2. `jetson.py`
* **역할**: 🚀 **클라이언트 프로그램**
* **주요 기능**:
    * 분할된 딥러닝 모델의 앞부분(`part1`)을 로드합니다.
    * 입력 이미지를 전처리하고, 모델의 앞부분 추론을 수행하여 중간 결과물(텐서)을 생성합니다.
    * 생성된 텐서를 `server.py`로 전송하고, 서버로부터 최종 예측 결과를 수신하여 출력합니다.
* **실행 방법**:
    ```bash
    python jetson.py
    ```

### 3. `model_utils.py`
* **역할**: 🛠️ **모델 처리 유틸리티**
* **주요 기능**:
    * `torchvision`에서 사전 훈련된 MobileNetV2 모델을 불러옵니다.
    * 지정된 레이어를 기준으로 모델을 두 부분(`part1`, `part2`)으로 분할하는 `get_split_mobilenet_v2` 함수를 포함합니다.
    * `server.py`와 `jetson.py` 양쪽에서 `import`하여 사용되는 공통 모듈입니다.

### 4. `measure_tensor.py`
* **역할**: 📏 **진단용 스크립트**
* **주요 기능**:
    * 모델이 특정 지점에서 분할되었을 때 생성되는 중간 텐서의 크기를 측정합니다.
    * 두 가지 크기를 측정하여 출력합니다:
        1.  **메모리상 크기 (MB)**: RAM에서 텐서가 차지하는 실제 용량
        2.  **직렬화 크기 (MB)**: 네트워크로 전송될 때의 데이터 패킷 크기
* **실행 방법**:
    ```bash
    python measure_tensor.py
    ```

---

## ⚙️ 실행 순서
1.  **서버 실행**: `server.py`를 먼저 실행하여 클라이언트의 요청을 기다리는 상태로 만듭니다.
2.  **클라이언트 실행**: `jetson.py`를 실행하여 서버에 추론을 요청하고 최종 결과를 받습니다.
